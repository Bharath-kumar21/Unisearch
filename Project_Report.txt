1. Introduction
The UniSearch India platform is a full-stack web application designed to help Indian students discover, compare, and shortlist engineering universities. The platform integrates various parameters such as entrance exam type, rank, location, fees, and more. This project is built utilizing a modern tech stack consisting of React for the frontend, Node.js/Express for the backend, and MongoDB for data storage and management.

1.1 Problem Statement
Currently, students face significant challenges in finding appropriate universities based on their specific entrance exam scores (like JEE Main, JEE Advanced, CUET, State CET, etc.). Information is scattered across multiple portals, and comparing colleges manually is tedious, error-prone, and overwhelming. There is a lack of a centralized, user-friendly platform that seamlessly filters universities by dynamic, cross-referenced criteria.



--- [Page 3 ended, jumped to Page 4] ---

1.2 Objective of the project
The primary objective is to develop a seamless and interactive web platform that aggregates university data. It aims to empower students to make informed decisions by providing accurate, up-to-date information on admissions, cutoffs, placements, and fees, along with a side-by-side comparison feature.



--- [Page 4 ended, jumped to Page 5] ---

1.3 Purpose of the project
The purpose is to democratize access to higher education information in India. By providing a single source of truth that is easily navigable, the platform reduces the stress of the admission process and ensures students can find institutions where they meet the criteria.



--- [Page 5 ended, jumped to Page 6] ---

1.4 Scope of the project
The scope is limited to engineering universities across India, encompassing both public and private institutions. Features include rank-based filtering, side-by-side comparison of up to three universities, data visualization for analytical insights, and user profile management for saving shortlisted universities.

1.5 Benefits of the project
Benefits include time savings for students and parents, improved accessibility to accurate university statistics, personalized shortlists via user accounts, and a transparent comparison mechanism that brings clarity to placement records and fee structures.

3. Software Requirement Analysis
The system requires a robust backend capable of handling complex aggregation queries. The frontend must be responsive to support both desktop and mobile users. Functional requirements include user authentication, complex filtering mechanisms, and data retrieval for visualization.

3.1 Domain Analysis
The educational domain in India is vast, with diverse entrance exams governed by different state and central authorities. The analysis covered the mechanisms of JEE, CUET, and various state-level CETs to structure a unified data model that accommodates these disparities.

4. System analysis Design
The system adopts a client-server architecture model. The React client delegates intensive data management to the Context API, whilst the Express server handles route protections and interacts asynchronously with the MongoDB instance using Mongoose ODMs.

4.1 System Design
The monolithic backend API interfaces with a Single Page Application (SPA). The communication happens over RESTful endpoints. The design incorporates a Context-based state management system for authentication and saved universities, ensuring smooth data flow.

5. Implementation and Testing
Implementation utilizes Node.js and React. Testing encompasses various facets structured to ensure reliability, optimal performance, and security across both API and user interface layers.

5.1 Implementation of Codes
The codebase follows a Modular pattern. The Node Server employs MVC principles separating routes, controllers, and models. The React client follows a component-based directory grouping by pages, services, and contexts.
